# Changelog

All notable changes to Iron Claude will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-11-08

### Added
- **Four Specialized Personas**:
  - Product Manager: UX validation, Hotwire patterns, feature completeness
  - DevOps Engineer: Kamal deployment, Solid Stack expertise, performance analysis
  - QA Tester: TDD enforcement, test coverage guardian, edge case hunter
  - Code Reviewer: Security auditor, Rails conventions enforcer, omakase advocate

- **Milestone-Based Review Gates**:
  - Automatic validation at key workflow points
  - LLM-based evaluation blocks incomplete work
  - Context-aware milestone tracking

- **TDD Workflow Support**:
  - Test-first enforcement
  - Automatic test execution on code changes
  - Coverage tracking (90%+ target)

- **Rails 8 Omakase Expertise**:
  - Hotwire/Turbo 8 patterns (morphing, frames, streams)
  - Solid Stack configuration (Queue, Cache, Cable)
  - Kamal 2 deployment best practices
  - Propshaft asset pipeline

- **Workflow Commands**:
  - `/review-feature`: Invoke all personas for comprehensive review
  - `/pre-deploy`: Pre-deployment checklist and validation
  - `/milestone-planning`: Feature milestone planning
  - `/security-audit`: Comprehensive security review
  - `/performance-check`: N+1 queries and performance analysis

- **Specialized Skills**:
  - Rails security audit with automated Brakeman integration
  - Performance analysis with query optimization
  - Hotwire patterns library with examples

- **GitHub Integration**:
  - PR review automation
  - Issue triage support
  - CI/CD awareness

- **Comprehensive Documentation**:
  - Getting started guide
  - Persona descriptions and usage
  - TDD workflows
  - DHH omakase philosophy
  - Troubleshooting guide

### Philosophy
Iron Claude embodies DHH's omakase philosophy: Convention over Configuration, Majestic Monolith, no PaaS required, database-backed simplicity, and developer happiness through compression of complexity.
